# Item Types

> Reference documentation for DevStash's 7 built-in (system) item types.
>
> Generated by `/research item-types-research.md`. Sources: `context/project-overview.md`,
> `prisma/schema.prisma`, `prisma/seed.ts`, `src/components/dashboard/type-icons.ts`,
> `src/lib/db/items.ts`.
>
> **Note:** the research prompt cited `src/lib/constants.tsx`, which does not exist.
> The equivalent constants are split across two files: the icon-name → Lucide component
> map in `src/components/dashboard/type-icons.ts`, and the canonical ordering
> (`TYPE_ORDER`) plus label derivation in `src/lib/db/items.ts`.

## Overview

An **item type** is a row in the `ItemType` table. The 7 built-in types are *system*
types: `isSystem = true` and `userId = null`, seeded by `prisma/seed.ts`
(`SYSTEM_ITEM_TYPES`) and not editable or deletable by users. Custom user types
(`isSystem = false`, `userId` set) are a planned Pro feature and are not yet seeded.

Every `Item` references exactly one `ItemType` (`itemTypeId`, `onDelete: RESTRICT`).
The type drives three things:

- **Icon** — the Lucide icon shown on cards and in the sidebar.
- **Color** — the accent color (card border tint, sidebar dot, collection card tint).
- **ContentType** — TEXT / FILE / URL, which decides *which* `Item` fields are used.

## ContentType Classification

The `ContentType` enum (TEXT / FILE / URL) is derived from the type name in
`seed.ts` (`contentTypeFor`) and the same mapping is the rule throughout the app:

| ContentType | Types | Primary content field | Null fields |
|---|---|---|---|
| **TEXT** | snippet, prompt, command, note | `content` | `fileUrl`, `fileName`, `fileSize`, `url` |
| **FILE** | file, image | `fileUrl` (+ `fileName`, `fileSize`) | `content`, `url` |
| **URL** | link | `url` | `content`, `fileUrl` |

Logic (from `seed.ts`):

```ts
function contentTypeFor(typeName: string): ContentType {
  if (typeName === "link") return ContentType.URL;
  if (typeName === "file" || typeName === "image") return ContentType.FILE;
  return ContentType.TEXT; // snippet, prompt, command, note
}
```

## Shared Properties

Every item — regardless of type — carries these `Item` fields:

- **Identity / display:** `title`, `description`, `language`
- **Organization:** belongs to a `User`, an `ItemType`, many `Collection`s (via
  `ItemCollection`), and many `Tag`s (via `ItemTag`)
- **Flags:** `isFavorite`, `isPinned`
- **Activity:** `lastUsedAt` (powers "Recently Used"), `createdAt`, `updatedAt`

These are type-agnostic: pinning, favoriting, tagging, collection membership, and
recency tracking work identically for a snippet, a file, or a link.

## Display Differences

The type is resolved to its presentation via two helpers:

- **Icon:** `getTypeIcon(iconName)` in `src/components/dashboard/type-icons.ts` maps
  the stored `ItemType.icon` string (`"Code"`, `"Sparkles"`, …) to a Lucide
  component, falling back to `File` for any unknown name.
- **Label:** `toLabel(name)` (in `src/lib/db/helpers.ts`) capitalizes the raw type
  name for display (`"snippet"` → `"Snippet"`); the sidebar appends an `s` for the
  plural nav label (`"Snippets"`) and builds the `href` as `/items/${name}s`.

Color usage:

- **Item cards** (`ItemCard.tsx`) — left border tinted in the type's color.
- **Collection cards** (`CollectionCard.tsx`) — top border / tint uses the
  `primaryColor`, i.e. the color of the collection's **most common** item type.
- **Sidebar** — recent collections show a colored dot tinted by their most-used type.

Pro gating:

- **file** and **image** show a muted `PRO` badge in the expanded sidebar
  (`SidebarContent.tsx`), and their upload is enforced server-side (`isPro` check),
  not just in the UI.

Field-driven rendering:

- **TEXT** types render `content` (Markdown for notes, syntax-highlighted code for
  snippet/command via `language`) with copy-to-clipboard.
- **FILE** types render a file/image preview from `fileUrl` with `fileName` /
  `fileSize` metadata; images get a thumbnail.
- **URL** types render `url` as a clickable link.
